<Heading>Theme System</Heading>

Pine UI provides **three distinct design themes** that transform your entire UI with a single prop change. Each theme has its own unique visual language, supporting both light and dark modes.

### üéØ Basic Theme

A clean, modern design system perfect for professional applications. Features smooth animations, subtle shadows, and a refined aesthetic.

<Source
	code={`import { ThemeProvider } from "pine-design-system";

function App() {
  return (
    <ThemeProvider design="basic" defaultTheme="light">
      <YourApp />
    </ThemeProvider>
  );
}`}
	language="tsx"
/>

### üéÆ Game Theme

Pixel-art inspired design for gamified experiences. Features retro aesthetics, bold borders, and playful interactions that bring a nostalgic gaming feel to your interface.

<Source
	code={`import { ThemeProvider } from "pine-design-system";

function App() {
  return (
    <ThemeProvider design="game" defaultTheme="light">
      <YourApp />
    </ThemeProvider>
  );
}`}
	language="tsx"
/>

### üñçÔ∏è Crayon Theme

Soft, warm, and friendly design with hand-drawn aesthetics. Features rounded corners, gentle shadows, and a cozy feel perfect for creative or educational applications.

<Source
	code={`import { ThemeProvider } from "pine-design-system";

function App() {
  return (
    <ThemeProvider design="crayon" defaultTheme="light">
      <YourApp />
    </ThemeProvider>
  );
}`}
	language="tsx"
/>

### Theme Provider Props

- **design** (optional): `"basic" | "game" | "crayon"` - The design variant to apply (default: `"basic"`)
- **defaultDesign** (optional): `"basic" | "game" | "crayon"` - Initial design for uncontrolled mode
- **onDesignChange** (optional): `(design: Design) => void` - Callback when design changes
- **defaultTheme** (optional): `"light" | "dark"` - Initial theme for uncontrolled mode
- **theme** (optional): `"light" | "dark"` - Controlled theme value
- **onThemeChange** (optional): `(theme: Theme) => void` - Callback when theme changes
- **syncWithSystem** (optional): `boolean` - Sync with system theme (default: `true`)
- **primaryColor** (optional): `string` - Custom primary color (hex format)

All three design variants support both light and dark themes.

You can enable automatic system theme detection:

	language="tsx"
/>

### Controlled Theme and Design

For complete control over both theme and design:

<Source
  code={`import { useState } from "react";
import { ThemeProvider } from "pine-design-system";

function App() {
  const [theme, setTheme] = useState<"light" | "dark">("light");
  const [design, setDesign] = useState<"basic" | "game" | "crayon">("basic");

  return (
    <ThemeProvider 
      design={design} 
      onDesignChange={setDesign}
      theme={theme} 
      onThemeChange={setTheme}
    >
      <div>
        <button onClick={() => setTheme(theme === "light" ? "dark" : "light")}>
          Toggle Light/Dark
        </button>
        <button onClick={() => setDesign("basic")}>Basic Design</button>
        <button onClick={() => setDesign("game")}>Game Design</button>
        <button onClick={() => setDesign("crayon")}>Crayon Design</button>
      </div>
      <YourApp />
    </ThemeProvider>
  );
}`}
language="tsx" />

<Heading>Design Tokens</Heading>

### Intent (Color Theme)

All components support these color intents:

- `primary` - Primary actions
- `secondary` - Secondary actions
- `success` - Success, completion
- `warning` - Warning, caution
- `danger` - Danger, deletion
- `neutral` - Neutral information

<Source
	code={`<Button intent="primary">Primary</Button>
<Button intent="success">Success</Button>
<Button intent="danger">Delete</Button>`}
	language="tsx"
/>

### Size

Most components support these sizes:

- `small` - Small
- `medium` - Medium (default)
- `large` - Large
- `xlarge` - Extra large

<Source
	code={`<Button size="small">Small</Button>
<Button size="medium">Medium</Button>
<Button size="large">Large</Button>`}
	language="tsx"
/>

### Variants

Components support different visual variants:

**Button variants:**

- `solid` - Filled button (default)
- `outline` - Outlined button
- `ghost` - Transparent button
- `weak` - Subtle button

**Badge variants:**

- `solid` - Filled badge (default)
- `outline` - Outlined badge
- `subtle` - Subtle badge
- `weak` - Weak badge

<Source
	code={`<Button variant="solid">Solid</Button>
<Button variant="outline">Outline</Button>
<Button variant="ghost">Ghost</Button>`}
	language="tsx"
/>

<Heading>Examples</Heading>

### Login Form

<Source
  code={`import { useState } from "react";
import { Button, TextField, Checkbox } from "pine-design-system";

function LoginForm() {
  const [email, setEmail] = useState("");
  const [password, setPassword] = useState("");
  const [remember, setRemember] = useState(false);
  
  return (
    <form>
      <TextField
        label="Email"
        type="email"
        placeholder="your@email.com"
        value={email}
        onChange={(e) => setEmail(e.target.value)}
        fullWidth
      />

      <TextField
        type="password"
        label="Password"
        placeholder="password"
        value={password}
        onChange={(e) => setPassword(e.target.value)}
        fullWidth
      />

      <Checkbox
        label="Remember me"
        checked={remember}
        onChange={(e) => setRemember(e.target.checked)}
      />

      <Button type="submit" intent="primary" fullWidth>
        Sign In
      </Button>
    </form>
  );
}`}
language="tsx"
/>
